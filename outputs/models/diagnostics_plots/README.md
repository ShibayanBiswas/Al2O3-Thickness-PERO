# Modeling Diagnostics Plots Index

This directory contains the full diagnostics set for every model and every target. Plots are organized into a hierarchy by model name, then by target name, and then by diagnostic category. This structure makes it easy to audit a single target across models or to audit a single model across all targets without mixing files in one flat folder. It also ensures that calibration plots, residual plots, and distribution checks remain separated so you can compare like with like.

The hierarchy is: `Model` → `Target` → `Calibration` or `Residuals` or `Distributions`. Calibration contains parity style plots and sorted actual versus predicted plots, which reveal how well the model aligns with observed values in sample. Residuals contains residual structure plots against predicted values, actual values, and thickness, plus absolute error versus thickness, which is particularly important in a discrete thickness design. Distributions contains residual histograms with density overlays, residual box plots, predicted versus actual density comparisons, and QQ plots.

Because metrics are in sample, a very high \(R^2\) can occur when a model effectively memorizes thickness cohorts. The residuals versus thickness plot is therefore treated as a primary diagnostic. A good model will show residuals that are centered without strong thickness dependent structure, while still producing a smooth and plausible response curve. If residuals cluster by thickness level, that is a warning sign of cohort memorization or of a threshold like structure that a simple curve cannot capture.

All figures share a consistent PERO visual language. Titles and labels are written in Title Case and avoid parentheses, while mathematical notation uses LaTeX style mathtext for scientifically standard symbols. Legends are moved outside the axes region so that they never overlap the data or shaded diagnostic bands. This directory is intended to be directly usable for report figures without further manual styling edits, and the folder structure is designed to map cleanly onto report subsections.
