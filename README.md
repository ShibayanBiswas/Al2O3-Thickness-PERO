# Al2O3 Thickness PERO Analysis (Single-Feature, Multi-Output)

This project produces a **research-grade PERO-style** (polished, export-ready, reproducible, organized) analysis of how **`Al2O3 Thickness_nm`** affects four battery-relevant outputs:

- `Rct_initial_ohm`
- `ICE_percent`
- `Initial Reversible Capacity_mAh_g at 0.1C`
- `Highest Capacity Retention_percent`

## Data rules (hard constraints)

- Data source: `Data/Data.xlsx`
- Use **only** the Excel sheet named `Dataset`
- Ignore the `Sample` column completely
- Input feature: **only** `Al2O3 Thickness_nm`
- Fit **final** models on **100% of the 51 rows** (no normal train/test split)
- Diagnostics/plots are **in-sample** unless explicitly labeled as secondary stability diagnostics

## Quickstart

### 1) Create environment and install dependencies

```bash
python -m pip install -r requirements.txt
```

If you have multiple Python installations on Windows, you may prefer:

```bash
py -m pip install -r requirements.txt
```

### 2) Run the full pipeline (EDA → Models → Diagnostics → Explainability → Exports)

```bash
py run_all.py
```

Outputs are written to `outputs/` (plots, tables, and report files).

## Deliverables

- **Notebook**: `notebooks/Al2O3_Thickness_PERO.ipynb`
- **One-command pipeline**: `run_all.py`
- **Postprocessing helper**: `postprocess.py`
- **Modular library code**: `src/`
- **All plots & tables**: `outputs/` with READMEs inside each folder

## Codebase Structure

This section inventories the full codebase folder by folder so you can navigate quickly and cite artifacts cleanly in a report.

### Root Directory

- **`README.md`**: Project overview, rules, and how to run the PERO pipeline.
- **`requirements.txt`**: Python dependencies. Optional libraries are included but the code will skip gracefully if not available.
- **`run_all.py`**: Main end to end pipeline. Deletes old `outputs/` and regenerates deep EDA, model suite results, diagnostics, explainability, and a summary report.
- **`postprocess.py`**: Finishes model comparison tables, explainability, and the summary report from already exported `metrics__*.csv` files.
- **`catboost_info/`**: Automatically generated by CatBoost when installed and used.

### Data

- **`Data/Data.xlsx`**: The only data source. The pipeline reads only the sheet named `Dataset`, ignores the `Sample` column, and treats thickness as the only feature.

### Source Code

All reusable functionality lives under `src/`.

- **`src/config.py`**: Project paths, column specification, and run configuration such as random seed and plot export settings.
- **`src/io_data.py`**: Excel loading, column standardization, strict numeric coercion, and dataset validation checks.
- **`src/audit.py`**: Compact audit table generation and thickness value concentration summaries.
- **`src/viz_style.py`**: PERO theme definition, high contrast matplotlib rcParams, label formatting utilities, and axis polishing helpers.
- **`src/plots.py`**: Plot saving helpers, consistent export behavior, and annotation utilities.
- **`src/eda.py`**: Deep EDA plot suite and table exports. Organizes plots into univariate, bivariate, grouped, and relationship hierarchies.
- **`src/models.py`**: Model suite definitions for multi output regression. Applies Standard Scaler where required and wraps models for multi output compatibility.
- **`src/model_eval.py`**: Metric computation per target, overall summaries, and ranking utilities for comparing models.
- **`src/diagnostics.py`**: In sample diagnostic plots with additional folder depth for calibration, residuals, and distribution checks.
- **`src/explainability.py`**: Permutation importance, PDP and ICE where supported, sensitivity style curves, and SHAP exports when compatible.
- **`src/report.py`**: Summary report writer and Markdown table rendering that does not depend on optional third party formatters.
- **`src/utils.py`**: Safe filename utilities, title casing, robust statistics helpers, and Excel export utilities.

### Notebook

- **`notebooks/Al2O3_Thickness_PERO.ipynb`**: A report style notebook wrapper that runs the pipeline and points to exported artifacts under `outputs/`.
